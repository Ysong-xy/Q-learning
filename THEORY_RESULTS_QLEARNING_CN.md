# 透明市场与不透明市场下的 Q-Learning 定价实验

本文档用中文系统说明本项目的理论逻辑、市场模型、Q-Learning 过程，以及透明市场和不透明市场两种情况下的实验结果。

项目研究的问题是：

> 在重复定价市场中，如果 Agent 只能看到自己的上一轮价格和收益，它们是否会回到竞争性 Nash 价格？如果 Agent 能看到所有人的上一轮价格和收益，是否更容易形成高于 Nash 的默契合谋价格？

当前代码中使用的是一个 4 市商重复定价模型。核心实现文件是：

- `repeated_market_rl.py`：市场环境、收益函数、状态定义、Q-learning 训练；
- `run_experiment.py`：运行实验、保存结果、调用画图；
- `plot_results.py`：生成训练曲线、最终价格对比、最终利润对比和价格分布图。

## 1. 理论问题

### 1.1 单轮市场中的 Nash 竞争

在单轮定价博弈中，每个市商只关心当前这一轮利润。假设价格只有两个选择：

```text
低价：2.0
高价：2.6
```

如果其他市商选择高价，那么某一个市商选择低价可以抢走更多需求，获得更高的短期收益。

如果其他市商选择低价，那么自己继续高价会失去需求，因此也应该选择低价。

所以在单轮市场中，低价是稳定的竞争性结果。代码中把这个结果称为 one-shot Nash：

```text
Nash price = 2.0
Nash profit = 410
```

这对应所有市商都低价：

```text
[0, 0, 0, 0]
```

其中 `0` 表示低价。

### 1.2 共同利润最大化结果

如果所有市商都选择高价，则每个市商的单位利润更高。虽然需求可能减少，但在当前模型中，总体利润仍然更高：

```text
Joint-profit price = 2.6
Joint-profit profit = 500
```

这对应所有市商都高价：

```text
[1, 1, 1, 1]
```

其中 `1` 表示高价。

注意，这不是单轮 Nash。因为只要其他人都高价，某个市商就有动机偷偷降价抢需求。

### 1.3 重复博弈中的默契合谋

如果市场不是只进行一轮，而是重复很多轮，那么市商不仅关心当前利润，还会关心未来利润。

这时高价结果可能通过一种隐含规则被维持：

```text
如果大家都维持高价，则继续高价；
如果有人降价，则其他人之后也降价惩罚；
如果惩罚后重新稳定，则尝试恢复高价。
```

这种机制不需要市商之间显式通信。只要所有人能观察到上一轮市场状态，它们就可以通过学习形成类似策略。这就是 tacit collusion，即默契合谋。

## 2. 不透明市场与透明市场的理论区别

本项目的核心对照是两种信息结构。

### 2.1 不透明市场

不透明市场中，每个市商只能看到自己的上一轮信息：

```text
自己的上一轮价格
自己的上一轮收益
```

它看不到其他市商上一轮是否高价、是否低价，也无法准确判断自己收益变化来自哪里。

例如，一个市商发现自己上一轮收益下降，原因可能是：

- 自己价格太高；
- 别人降价抢走需求；
- 随机噪声影响了观察到的收益；
- 市场进入了混合状态。

由于缺少公共状态，不透明市场中的 Agent 很难建立“谁偏离了，所以我要惩罚”的策略。它更像是在学习一个局部最优反应：在不确定别人行为的情况下，低价更稳妥。

因此理论预期是：

```text
不透明市场 -> 更接近 Nash 低价
```

### 2.2 透明市场

透明市场中，每个市商都能看到所有市商上一轮的动作：

```text
上一轮所有人的价格选择
```

例如：

```text
[1, 1, 1, 1] 表示上一轮所有人高价
[0, 0, 0, 0] 表示上一轮所有人低价
[0, 1, 1, 1] 表示有一个市商降价
```

这使得上一轮价格向量成为公共历史。所有 Agent 都看到同一个公共状态，因此可以学习条件策略。

透明市场中的 Agent 可以把不同状态区分开：

```text
全高价状态：维持高价
混合状态：说明有人偏离，进入低价惩罚
全低价状态：惩罚结束后尝试恢复高价
```

因此理论预期是：

```text
透明市场 -> 更容易维持高价协调 -> 价格高于 Nash
```

这就是本项目要展示的主要现象：市场透明度提高后，竞争未必更强，反而可能使市商更容易通过公共历史形成默契合谋。

## 3. 市场模型

### 3.1 基本参数

当前模型使用 4 个市商：

```python
n_market_makers = 4
```

价格只有两个动作：

```python
low_price = 2.0
high_price = 2.6
```

初始状态设置为：

```python
initial_actions = (0, 0, 1, 1)
```

也就是两个市商低价，两个市商高价。因此初始平均价格为：

```text
(2.0 + 2.0 + 2.6 + 2.6) / 4 = 2.3
```

这样初态正好位于 Nash 价格 `2.0` 和 joint-profit 价格 `2.6` 之间，避免实验结果被“从低价开始”或“从高价开始”主导。

### 3.2 收益结构

收益函数定义在 `repeated_market_rl.py` 的 `demand_and_reward` 函数中。主要参数是：

```python
nash_profit = 410.0
collusive_profit = 500.0
temptation_profit = 560.0
sucker_profit = 100.0
```

含义如下。

| 市场状态 | 单个市商收益含义 |
| --- | --- |
| 所有人低价 | 每个市商获得 Nash 收益 410 |
| 所有人高价 | 每个市商获得共同利润收益 500 |
| 混合状态中的低价者 | 获得抢市场收益，最高接近 560 |
| 混合状态中的高价者 | 被低价者抢走需求，收益较低 |

这个收益结构保证了重复定价博弈的基本矛盾：

```text
短期看，降价有诱惑；
长期看，所有人高价更好。
```

### 3.3 从收益反推需求

模型中利润满足：

```text
profit = (price - cost) * demand
```

代码中先设定收益，再根据价格反推出需求：

```python
demand = rewards / (price_vector(actions, cfg) - cfg.cost)
```

这样做的原因是：本项目重点不是估计真实消费者需求，而是展示重复博弈中“公共信息如何支持高价协调”的机制。因此使用 reduced-form payoff 会让机制更清楚。

## 4. Q-Learning 过程

### 4.1 Q-Learning 的目标

每个市商都是一个独立的 Q-learning Agent。它的目标是学习一个策略：

```text
在当前状态 s 下，选择哪个动作 a 能带来更高长期收益
```

Q 值定义为：

```text
Q_i(s, a)
```

含义是：对第 `i` 个市商来说，在状态 `s` 下选择动作 `a` 的长期价值。

动作空间是：

```text
a = 0 -> 选择低价 2.0
a = 1 -> 选择高价 2.6
```

### 4.2 状态定义

两种市场的状态定义不同，这是实验最核心的部分。

#### 不透明市场状态

不透明市场中，Agent `i` 的状态是：

```python
(自己的上一轮动作, 自己上一轮收益区间)
```

代码实现：

```python
if mode == "private":
    return (int(actions[agent]), _reward_bin(float(observed_rewards[agent])))
```

这里 `_reward_bin` 把连续收益离散成区间。例如收益不同会被映射到不同 bin 中，使 Q 表可以处理有限状态。

不透明市场的状态没有包含其他市商动作。因此，即使市场中有人降价，某个 Agent 也不能直接知道是谁降价、降了多少、是否所有人都看到了同样的事件。

#### 透明市场状态

透明市场中，Agent 的状态是完整上一轮动作向量：

```python
return tuple(int(action) for action in actions)
```

例如：

```text
(1, 1, 1, 1)
(0, 0, 0, 0)
(0, 1, 1, 1)
```

这个状态是公共的。所有 Agent 都能看到同一个上一轮动作向量，因此它可以作为协调信号。

### 4.3 动作选择：epsilon-greedy

每一轮，每个 Agent 根据 epsilon-greedy 选择动作：

```text
以 epsilon 的概率随机探索；
以 1 - epsilon 的概率选择 Q 值最大的动作。
```

代码逻辑是：

```python
if rng.random() < epsilon:
    actions[i] = rng.integers(0, 2)
else:
    actions[i] = argmax_a Q_i(state, a)
```

探索率随训练衰减：

```python
epsilon = epsilon_end + (epsilon_start - epsilon_end) * exp(-t / epsilon_decay)
```

当前参数为：

```python
epsilon_start = 0.04
epsilon_end = 0.005
epsilon_decay = 20000
```

这表示早期有少量随机尝试，后期逐渐稳定。

### 4.4 Q 值更新

一轮训练过程如下：

1. 根据上一轮市场状态构造当前状态 `s_i`；
2. 每个 Agent 选择动作 `a_i`；
3. 市场根据所有动作计算收益 `r_i`；
4. 根据新动作和新收益构造下一状态 `s_i'`；
5. 用 Q-learning 更新 Q 表。

标准 Q-learning 更新目标是：

```text
target = r_i + gamma * max_a Q_i(s_i', a)
```

TD error 是：

```text
TD error = target - Q_i(s_i, a_i)
```

然后更新：

```text
Q_i(s_i, a_i) <- Q_i(s_i, a_i) + alpha * TD error
```

代码中对应为：

```python
target = rewards[i] + train_cfg.gamma * next_q.max()
td_error = target - q[actions[i]]
q[actions[i]] += learning_rate * td_error
```

折扣因子是：

```python
gamma = 0.99
```

这表示 Agent 非常重视未来收益。这个参数对默契合谋很重要，因为如果 Agent 只看当前收益，降价抢市场通常更有吸引力。

### 4.5 Hysteretic Q-Learning

当前代码使用了 hysteretic Q-learning，即正向 TD error 和负向 TD error 使用不同学习率：

```python
learning_rate_positive = 0.05
learning_rate_negative = 0.001
```

代码：

```python
learning_rate = (
    train_cfg.learning_rate_positive
    if td_error >= 0
    else train_cfg.learning_rate_negative
)
```

含义是：

```text
当结果比预期好时，快速学习；
当结果比预期差时，慢速学习。
```

为什么要这样做？

在多 Agent 环境中，一个 Agent 的收益下降不一定说明自己的策略不好，也可能是其他 Agent 正在探索。比如透明市场中，大家本来维持高价，但某一轮某个 Agent 随机探索低价，其他高价 Agent 的收益会下降。如果所有 Agent 立刻把“高价策略”判断为失败，就很难维持长期合作。

Hysteretic Q-learning 让 Agent 对偶发负面结果不那么敏感，因此更适合重复博弈和多 Agent 协调场景。

## 5. 策略先验

代码中有一个 Q 值初始化先验：

```python
repeat_last_prior = 10000.0
```

它只在某个状态第一次被访问时使用，用于给某个动作一个较高初始 Q 值。

### 5.1 不透明市场中的先验

不透明 Agent 只知道自己的上一轮动作，因此它的先验是：

```text
倾向于重复自己的上一轮动作
```

代码：

```python
previous_action = state[0]
q[previous_action] = train_cfg.repeat_last_prior
```

这是一种非常弱的行为惯性，不包含别人动作信息。

### 5.2 透明市场中的先验

透明 Agent 能看到完整公共状态，因此先验可以依赖上一轮市场结构：

```python
if all(action == 1 for action in state) or all(action == 0 for action in state):
    q[1] = train_cfg.repeat_last_prior
else:
    q[0] = train_cfg.repeat_last_prior
```

含义是：

- 如果上一轮所有人高价，则尝试继续高价；
- 如果上一轮所有人低价，则尝试恢复高价；
- 如果上一轮是混合状态，则认为有人偏离，进入低价惩罚。

这一步非常关键。它并不是显式写死最终策略，因为 Q 值仍然会通过收益反馈更新；但它让透明 Agent 能够表达一种基于公共历史的策略结构。不透明 Agent 因为没有公共历史状态，无法表达同样的规则。

## 6. 两种情况的学习逻辑

### 6.1 不透明市场的学习逻辑

不透明市场中，Agent 看到的是：

```text
我上一轮选了什么？
我上一轮赚了多少？
```

它看不到：

```text
别人上一轮是否降价？
有几个人降价？
当前是否处于惩罚阶段？
大家是否已经恢复高价？
```

所以它很难区分：

```text
我是因为自己高价而亏？
还是因为别人暂时探索低价而亏？
```

在这种情况下，低价是更稳妥的动作。训练足够久后，不透明组最终倾向于：

```text
所有市商低价
平均价格接近 2.0
利润接近 Nash profit = 410
```

### 6.2 透明市场的学习逻辑

透明市场中，Agent 看到的是完整上一轮动作向量：

```text
[a_1, a_2, a_3, a_4]
```

它可以区分：

```text
上一轮是否所有人高价？
上一轮是否有人降价？
上一轮是否所有人低价？
```

因此它可以学习更复杂的重复博弈策略：

```text
全高价 -> 继续高价
混合状态 -> 低价惩罚
全低价 -> 恢复高价
```

这种策略使高价状态可以被维持。因为偏离者虽然当前可能获得更高收益，但偏离会改变公共状态，触发之后的低价惩罚，从而降低长期收益。

训练足够久后，透明组最终倾向于：

```text
所有市商高价
平均价格接近 2.6
利润接近 joint-profit profit = 500
```

## 9. 结论

本实验支持如下结论：

1. 在不透明市场中，Agent 无法观察公共历史，难以识别和惩罚偏离行为，因此学习结果接近单轮 Nash 低价。
2. 在透明市场中，Agent 可以观察所有人的上一轮动作，公共历史成为可学习状态，因此能够形成维持高价的重复博弈策略。
3. 市场透明度提高不一定增强竞争；在重复定价环境中，它可能降低协调难度，使默契合谋更容易出现。

简而言之：

```text
不透明信息 -> 局部学习 -> Nash 低价
透明信息 -> 公共历史 -> 惩罚与恢复 -> 高价协调
```

## 10. 需要注意的地方

当前模型是一个机制展示模型，而不是完整的真实市场估计模型。

它做了几个简化：

- 价格只有两个档位；
- 收益函数是 reduced-form payoff；
- 市商数量固定为 4；
- 使用 Q 表而不是深度神经网络；
- 只展示一个默认随机种子下的训练结果。

这些简化使理论机制更清楚，也让实验图表更稳定。后续如果要做更正式的研究，可以扩展为：

- 多价格档位；
- 连续价格动作；
- logit demand 消费者需求；
- 多随机种子平均结果；
- 不同透明度水平的中间情形；
- DQN、PPO 或多 Agent RL 算法。

但对于课程项目或方法展示，当前模型已经能清楚说明：透明信息通过公共历史改变了 Agent 的可学习策略空间，从而可能导致高于 Nash 的默契合谋价格。
